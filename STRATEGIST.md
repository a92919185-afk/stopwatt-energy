# STRATEGIST — Pre-Writing Planning Framework
## Think Before You Write. Plan Before You Build.

> **The principle behind this document:**
> A page written without a strategy is a page that guesses.
> A page written with a strategy knows exactly who it is talking to,
> what they fear, what they want, and precisely what needs to happen
> in each section for them to click the CTA.
>
> This phase uses three skills together:
> `copywriting` · `marketing-psychology` · `copy-editing`
>
> Run it BEFORE writing any JSON. Output the Strategy Brief BEFORE Phase 2.
> The brief IS the thinking. The JSON is just the execution of the brief.

---

## WHEN TO RUN THE STRATEGIST

| Trigger | Action |
|---------|--------|
| Starting any new page (any model) | Full Strategist brief required |
| Rewriting copy on an existing page | Run Parts 3–7 minimum |
| Owner says "the copy feels weak" | Run the full brief, then rewrite |
| Before running the REVIEWER (REVIEWER.md) | Brief must exist |
| Generating a country variant | Re-run Parts 3 and 5 for the new country |

---

## THE STRATEGY BRIEF FORMAT

The output of this phase is a single Strategy Brief document.
Save it as: `research/SLUG-strategy.md`
It becomes the ONLY permitted source for writing the JSON copy fields.

**The brief has 8 sections. All 8 are required. No section can be skipped.**

## PART 1 — PAGE IDENTITY

Before any audience analysis, lock the three fundamentals.

```

```

## PART 2 — THE AUDIENCE PSYCHOGRAPHIC PROFILE

> Demographics tell you who they are. Psychographics tell you why they buy.
> This section uses the `marketing-psychology` skill.

Answer all six questions. These answers become the invisible foundation of every copy decision.

### 2.1 — WHO IS THIS PERSON AT 11PM?

Describe the specific moment when this person searches for this product.
Not age or income. The moment. The feeling. The context.

```
Template:
"It's [time/situation]. [Name — give them a name, make them real].
He/she has just [specific triggering event]. He/she feels [emotion — not vague, specific].
He/she opens Google and searches [exact search query]. What he/she is really
asking is: [the underlying fear or desire behind the search]."
```

Example (Male Performance, US):
```
"It's 10:30pm. Marcus, 48, is lying in bed. His wife has fallen asleep.
Again. He feels ashamed — not at her, at himself. He opens his phone
and searches 'male performance supplement that actually works.'
What he's really asking is: 'Is there still a way back to who I was?'"
```

**Why this matters:** Every headline, every benefit line, every FAQ answer is written for THIS person in THIS moment. If you cannot picture them, you will write for everyone — which means you write for no one.

### 2.2 — THE JOBS-TO-BE-DONE (JTBD)

For this specific product + audience, fill in the four job types.

```
Functional job:  "I want to __________ so that __________."
                 (What they want the product to literally do)

Emotional job:   "I want to feel __________ again."
                 (The internal state they're buying toward)

Social job:      "I want [others] to __________ me."
                 (How they want to be seen by others)

Anxiety job:     "I'm afraid I'll __________ if I don't fix this."
                 (The fear driving urgency — not the desire)
```

**The anxiety job is the most powerful for BOFU copy.** It creates urgency without fake scarcity.
The anxiety job belongs in: the benefits section (paragraph 2) and the FAQ (Q1 and Q4).

### 2.3 — FAILED SOLUTIONS HISTORY

What has this person already tried that didn't work?
Extract this from Phase 1C research (Reddit, Amazon reviews, forums).

```
Attempted solution 1: [What they tried] → Why it failed: [Specific frustration]
Attempted solution 2: [What they tried] → Why it failed: [Specific frustration]
Attempted solution 3: [What they tried] → Why it failed: [Specific frustration]
```

**Why this matters:** BOFU copy must acknowledge the prior failures before claiming to solve the problem.
A visitor who has tried 3 things has earned their skepticism. Dismissing that skepticism loses them.
Naming their failures (without mocking them) is the fastest way to say "I understand you."

### 2.4 — THE OBJECTION MAP

List the 5 most likely objections to buying, in order of frequency.

```
Objection 1 (most common): "__________"
  → Where to neutralize it on the page: [Section + specific answer]
  → Psychology trigger to use: [Social proof / Authority / Regret aversion / etc.]

Objection 2: "__________"
  → Where: [Section]
  → Trigger: [Trigger]

[Repeat for 3, 4, 5]
```

Common supplement objections, ranked by BOFU frequency:
1. "Supplements never work for me" → neutralize in: hero subtext + FAQ Q1 + guarantee
2. "I can't tell if this is a real company" → neutralize in: trust badges + footer
3. "I don't know what's in it / if it's safe" → neutralize in: features + FAQ Q2
4. "It's too expensive / not worth it" → neutralize in: pricing/benefits CTA + FAQ Q6
5. "What if it doesn't work?" → neutralize in: FAQ Q4 (guarantee must be crystal clear)

### 2.5 — THE BELIEF SHIFT

State the ONE belief change the page must create.

```
The visitor arrives believing: "__________."
The visitor must leave believing: "__________."
```

Example:
```
Arrives: "My metabolism is just slow. Nothing I try makes a real difference."
Leaves:  "The reason nothing worked before is hormonal — and this product targets
          exactly that mechanism. The guarantee means I have nothing to lose trying."
```

**Every section of the page must push the visitor from the left column to the right column.**
If a section does not advance the belief shift, it is decorative — cut or rewrite it.

### 2.6 — THE SINGLE MOST IMPORTANT THING

Of everything you know about this audience and this product, what is the ONE thing
that will make or break this page's conversion?

```
The ONE thing: "__________"
```

Examples:
- "The target audience has tried 3+ supplements and failed. The hero must acknowledge that before anything else or they bounce."
- "The product's guarantee is 90 days — longer than any competitor. The guarantee must appear in 4 places: hero, benefits CTA, FAQ, footer CTA."
- "The traffic source is cold Instagram ads. The hero must establish what category this is in the first 5 words or visitors won't even read."

**This single sentence guides every copy decision for the rest of the brief.**

## PART 3 — PSYCHOLOGY TRIGGER MAP

> Use the `marketing-psychology` skill for this section.
> Map exactly which psychological mechanisms to activate in each section.

For this specific product + audience, fill in the trigger for each section.
Use the mechanism name AND state HOW it manifests in the copy.

```
Section          Trigger(s) to activate       How it appears in copy
─────────────────────────────────────────────────────────────────────
Alert bar        Scarcity + Social proof       "[X]% off today" + "[X,000]+ customers"
Hero             Loss aversion + Specificity   Name the cost of inaction + specific number
Trust badges     Authority bias                Specific certifications, not generic claims
Social proof     Bandwagon + Availability      "X,000 customers" makes success feel common
Benefits P1      Confirmation bias             Mirror their exact language from Reddit/reviews
Benefits P2      Regret aversion + JTBD        Name the emotional consequence of inaction
Benefits P3      Hope + Differentiation        Why THIS product, not just any product
Features         Authority + "Prove it"        Ingredient names + dosages + "which means"
Science          Cognitive ease                Complex → simple: 3 steps, plain language
Reviews          Mimetic desire                Reviewer = mirror of visitor's profile
FAQ              Uncertainty reduction         Every real objection, fully resolved
CTA              Zero risk + Present bias      Guarantee + "now" language
```

**Do not skip any row. Do not write "standard" — identify the specific application for THIS product.**

## PART 4 — THE EMOTIONAL ARC

> The page is a journey, not a list of features.
> Map the visitor's emotional state at each section, entering and exiting.

```
Section        Emotion ENTERING this section   What happens here         Emotion EXITING
───────────────────────────────────────────────────────────────────────────────────────
Alert bar      Neutral/scanning                Offer lands. Urgency felt. Mild interest
Hero           Skeptical/curious               Pain named. Hope offered.  "This is for me"
Trust          50% convinced                   Credibility established    Doubt reduced
Social proof   Cautiously interested           Bandwagon activated        "Others like me did this"
Benefits       Engaged but still skeptical     Pain agitated. Root cause. "Yes, exactly."
Features       Ready to learn why              Proof given. Science shown. Confidence builds
Reviews        Intellectually convinced        See themselves in reviews  "If it worked for them..."
FAQ            Almost decided, 1 objection     Objection fully resolved   "Nothing stopping me"
CTA            Ready                           Guarantee = zero risk      Clicks
```

**The emotional arc is broken if:** the visitor hits any section without the emotion listed in the "ENTERING" column. Fix the previous section, not the section where the break happens.

## PART 5 — COPY DIRECTION PER SECTION

> This is the brief for each copy element. Not the copy itself — the strategy for the copy.
> The JSON is written from this brief, not from generic formulas.

For each section, write 2–3 sentences of copy direction that are SPECIFIC to this product and audience.

```
ALERT BAR
Direction: [What offer/urgency is real and verifiable? What social proof stat to use?]
Avoid: [What tone or claim would feel fake for this specific audience?]

HERO HEADLINE
Direction: [Which formula from COPY.md / MODEL-B.md? Which emotion leads — loss aversion or hope?
            What specific word or phrase from the audience's real language should be in the headline?]
Avoid: [What generic phrases would make this visitor dismiss it as another fake ad?]

HERO SUBTEXT
Direction: [Subtext 1 — acknowledges what exactly? Subtext 2 — differentiates how?
            What's the ONE verifiable claim that must be in subtext 2?]

BENEFITS SECTION
Direction: [What is the root cause to name in P1? What's the emotional consequence in P2?
            What specific ingredient/mechanism is the bridge in P3?]
The "yes, exactly" moment: [The specific sentence that will make the visitor feel understood]

FEATURES
Direction: [Which 2–3 ingredients are the most persuasive for this audience?
            What "which means" bridge is most powerful for this specific audience?]
Lead ingredient: [Most important ingredient for this audience — lead with this one]

SCIENCE
Direction: [What 3-step mechanism is most credible and simple for this audience?
            What language level — more clinical or more accessible?]

REVIEWS
Direction: [What profile/demographics should each reviewer mirror?
            What specific outcome word must appear in at least one review?]
Must-have outcome: [The one specific result that this audience is most hungry for]

FAQ
Direction: [What are the top 3 objections specific to this product?
            Which objection needs the longest answer?]
Critical Q: [The question that, if answered poorly, kills the sale]

CTA
Direction: [Which CTA formula? What guarantee language? What "zero risk" framing?]
```

## PART 6 — COMPETITIVE DIFFERENTIATION ANGLE

State the single sharpest reason to choose this product over the alternative.

This must be ONE real, verifiable differentiator from the official product page.
Not a list of 5 things. ONE sharp point.

```
The differentiation angle: "__________"
Evidence: [Where on the official page is this supported?]
Where it appears on the page: [Hero? Benefits P3? Features lead card?]
```

**If you cannot find a real differentiator:**
Fall back to the experience differentiator:
- Guarantee length (if yours is longer than the category standard)
- Support quality (if the company has documented response times)
- Trust credentials (if certifications are more specific/credible than average)

Never invent a differentiator. The brief is not a place for optimism — it is a place for truth.

## PART 7 — QUALITY COMMITMENT

Before writing the JSON, sign off on the quality standard for this specific page.

```
Minimum acceptable standard for this page:

1. The hero headline must make the target audience say __________ (not "nice", something specific)
2. The benefits section must reference __________ (specific real pain from Phase 1C research)
3. Every feature card must answer "which means __________" specific to [audience's daily life]
4. At least one review must mirror __________ (specific profile from Section 2.1)
5. FAQ Q__ is the most critical objection — the answer must be __________ (how complete/what tone)
6. The CTA must include __________ (specific guarantee/risk language)
```

**Do not start writing until you have signed off on this section.**
If you cannot fill this section, you do not understand the audience well enough yet.
Go back to Phase 1 research.

---

## PART 8 — BRIEF SELF-AUDIT

Before handing the brief to Phase 2 writing, run this 5-question check:

1. **Specificity test:** Could any of the audience profile answers apply to a different product in the same niche? If yes, they are too generic. Rewrite them.

2. **"Yes, exactly" test:** Read the benefits copy direction aloud. Does it describe a real person's actual experience — or a marketing writer's version of it? If it sounds like marketing, it needs more research.

3. **Objection coverage test:** Does the brief address all 5 objections from Section 2.4? Every objection must have a section + trigger assigned. No objection can be left without a resolution point.

4. **Differentiation test:** If someone read only the differentiation angle (Part 6), would they understand why to choose this product over the first alternative they find on Google? If no, the differentiation is too vague.

5. **Belief shift test:** Read the page flow implied by the emotional arc (Part 4). Does it move the visitor from the left belief column to the right? Is there any section where the visitor could get stuck and stop reading? Fix that section's copy direction before writing.

If all 5 pass → proceed to Phase 2 (write the JSON).
If any fails → fix that section of the brief before proceeding.
